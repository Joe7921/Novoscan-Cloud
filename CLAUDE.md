# 项目规则

> 本文件每次会话自动加载。保持精简,详细计划和上下文放在独立文件中并在下方引用。

## 语言与交互
- 无论我使用什么语言提问,始终使用简体中文回复、写代码注释和任务说明。
- 当我的描述不清晰或执行中遇到需要人为决策的关键节点,必须停下来追问,严禁自动脑补需求细节。若我提出宏观方向,先给出两个你深度思考后最合适的细化方向建议,让我选择后再执行。
- 在完成我选定的优化方向后,若你认为还有其他值得做的优化,按价值从高到低至少提供三条建议。
- 非必要不硬编码。需要硬编码内容前,先告知我并说明优劣,由我决定。

## 工作区规则
- 当前目录是唯一可写区域,所有新代码、文档、配置都写在这里。
- D:\Antigravity projects\novoscan-next 是旧代码库,仅供参考和阅读,绝对禁止修改。
- 重构时参考旧代码的实现逻辑,在新工作区里重写,不要原样复制粘贴。

## 操作安全（红线规则）
1. 严禁在未获得我明确授权的情况下修改项目的整体视觉外观、CSS 样式、UI 组件库配置或设计风格。
2. 执行代码修改时严格限制在当前任务范围内。禁止擅自重构、修改或删除与当前需求无关的任何内容。
3. 可能影响项目全局架构或外观的操作,必须先提供方案并问我"是否同意执行",确认后方可操作。
4. 涉及删除文件、强推 Git、改部署配置等不可逆操作时,必须停下来等我确认。

## 文件编辑规则
1. 使用 Edit 和 Write 工具修改文件,不要用 shell 命令修改源代码文件。
2. 超过 500 行的文件先创建骨架再分段填入。
3. 影响超过 1 个文件的操作,先在单个文件上验证后再批量执行。
4. 关键文件修改完成后,立即运行构建验证结果再进行下一步。

## Shell 环境
当前环境为 Windows PowerShell。用 ; 分隔命令,不用 &&；用 $env:VAR="val" 设环境变量；用 Remove-Item -Recurse -Force 删目录。Edit/Write/Read/Grep/Glob 工具不经过 shell,不受此限制。

## 计划与上下文管理
所有计划和进度集中在 docs/ 目录:
- docs/REFACTOR-PRD.md ← 重构 PRD
- docs/PLAN.md ← 分阶段执行计划
- docs/PROGRESS.md ← 实时进度跟踪

规则:
1. 每完成一次计划设计,详细写入 docs/PLAN.md。
2. 每推进一步,更新 docs/PROGRESS.md（已完成、进行中、下一步、问题与决策记录）。
3. 新 session 开局先读 PROGRESS.md 再读 PLAN.md,然后继续工作。
