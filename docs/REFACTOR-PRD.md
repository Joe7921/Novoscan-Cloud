# Novoscan Cloud 重构 PRD（需求规格）

> 状态：**v1.0 已确认（2026-06-09）**
> 本文档面向非技术读者，每节先用大白话说清「做什么」，再补技术细节。
> 事实基础见 `docs/OLD-CODEBASE-ANALYSIS.md`。

---

## 0. 一句话目标

把现有的 Novoscan 产品，在新工作区里**从零全栈重写**成一个**网页应用**。

**产品定位（经两轮深聊确认）**：一个面向**专业人群**（创业者/PM、科研/技术、投资/分析师）的**"尽调级"创新分析工具**。用户**贴入想法 / 长文 / 文档 / 链接** → 系统**学术+产业双轨检索 + 多 AI 专家深度分析（辩论/仲裁/质检）** → 产出一份**经得起追问的可信报告**：每个判断**有出处可溯源**、摊开**反方观点与不确定性**、用**量化评分 + 多维雷达图**说话。

第一阶段只做最核心的一条主线（贴入文字 → 检索 → 多 Agent 分析 → 可信报告）。参考旧代码的业务逻辑，但代码全部重写，结构更清晰。

> **核心原则·可信度优先**：旧库的"证据溯源 / 辩论 / 仲裁加权透明 / 不确定性"在本产品里**不是后台细节，而是核心卖点**，报告中要做成显性功能。

---

## 0.5 终极形态（长远蓝图）

产品终极形态是一座**「线上办公楼」**：每层一个职责清晰的专业业务，共享同一个"分析大脑"（管线引擎）。详见 **`docs/ARCHITECTURE.md`**。

| 楼层 | 职责 |
|---|---|
| **Playground** | 分析主操作台：选不同行业定制**管线**（可热插拔/接市场/用户自制）；含 **Agentic Mode**（ReAct 智能体全自主动态适配分析） |
| **Flip** | 批量过项目（投资人等）：后台异步批量跑 → 批量"体检单"报告 + 问答 |
| **Project Base** | 项目迭代工作区（类比 Claude Code 工作区） |
| **Studio** | 用户创作自己的管线/报告 |
| **Marketplace** | 插件市场：收录与分发用户设计品 |

> 旧库的 Bizscan/Matchscan/Clawscan **未来不再是独立产品，而是 Playground 的预置管线**。
> **本期 = 先盖"第一层"**：管线引擎 + Playground 首条管线（即下文 §2 的核心分析闭环）。施工顺序与技术栈见 `docs/PLAN.md`、`docs/TECH-STACK.md`。

## 1. 为什么重写（背景）

旧产品 `novoscan-next` 功能强大但代码庞杂：155 个前端组件、近 90 个接口、新旧两套报告渲染并存、产品线交织。本次重写的目的：
- **理清并保留**真正有价值的核心业务逻辑（多 Agent 分析引擎、双轨检索、报告呈现）。
- **甩掉**历史包袱，用更清晰的结构重建，方便以后维护和扩展。
- 先把**核心闭环**跑通，再分阶段把其他产品线搬回来。

## 2. 本期范围（做什么 / 不做什么）

### ✅ 本期要做（核心分析闭环）

| 模块 | 大白话 | 技术要点 |
|---|---|---|
| 首页输入 | 用户贴入创意/长文（本期只标准深度模式）；文档上传/链接解析作为本期增量子阶段 7.5 | 富输入区（长文先行）+ 提交 |
| 实时分析过程 | 提交后能看到「AI 正在思考」的实时进度，而不是干等 | SSE 流式推送，展示各 Agent 状态/思考片段/进度条 |
| 双轨检索 | 自动去学术界（4 个论文库）和产业界（多个搜索引擎/GitHub）找资料 | 学术四源 + 产业多源，去重聚合 |
| 多 Agent 分析引擎 | 多个 AI「专家」分头分析，再有「评估师/仲裁员」汇总，必要时「辩论」 | 重写 orchestrator 分层调度 + ai-client 多模型容错 |
| 报告呈现 | 一份清晰报告：综合评分、6 维雷达图、关键发现、相似论文、证据来源、分析透明化 | 单一报告渲染引擎（不再新旧双版本） |
| 结果缓存 | 相同创意短期内重复查直接返回，省时间省成本 | Supabase 缓存最近结果 |

### ❌ 本期不做（后续阶段再加）

- 其他产品线：Bizscan / Matchscan / Clawscan / NovoMind / NovoDNA / NovoTracker / 插件市场 / CaseVault / 趋势。
- 用户登录、订阅付费、钱包积分、配额扣费（**本期先做成无需登录即可试用**，账号体系列入第二阶段）。
- MCP 开放接口、admin-cli 管理工具、Rust WASM 加速（先用普通代码实现质检，WASM 作为后续性能优化）。
- 追问精化（followup）—— 列入第二阶段。

> 说明：本期不做不等于永远不做，而是为了先把主干跑通。这些都会在 `PLAN.md` 里排进后续阶段。

## 3. 用户使用流程（站在用户角度）

1. 打开网址，看到首页输入框。
2. 输入自己的创意（如「固态电池新型电解质」），可选所属领域、选「极速」或「标准」模式。
3. 点「检查创新性」。
4. 页面进入分析中：实时看到「学术审查员、产业分析员、竞品侦探……」逐个工作的状态和思考片段，有进度条。
5. 分析完成，跳到报告页：
   - 顶部：综合创新评分 + 一句话结论（如「推荐（65 分）——该方向…」）。
   - 6 维创新雷达图（技术突破/应用契合/竞争态势/可实现性/盈利性/可持续性）。
   - 关键发现、风险红旗、相似论文（可点开跳原文）、证据来源。
   - 分析透明化：各专家评分的加权过程、是否触发辩论、少数派异议。
   - 可展开看原始检索数据。
6. （本期可选）报告可分享为公开链接。

## 3.5 界面与信息架构（核心原则：Claude Desktop 式分板块）

> 大白话：整个产品像 Claude Desktop 那样——**左侧一栏清晰的功能板块**，每个板块对应一件职责单一、任务明确的事；点哪个板块，主区域就专注做那件事。不再用旧版那种层层展开的"文件夹式"复杂导航。

设计要求：
1. **左侧板块导航栏**：固定在左侧，列出各功能板块。本期只有「创新分析」一个板块是激活的；其余产品线（Bizscan/人格评测/监控等）以"敬请期待"占位灰显，让用户一眼看清产品全貌和成长方向。
2. **每个板块 = 一个清晰任务**：进入板块后，主区域只聚焦该板块的事，任务目标、输入、输出都清晰，不堆砌无关信息。
3. **主区域专注**：核心分析板块的主区域承载「输入 → 分析中 → 报告」三态流转，干净、聚焦、一次只让用户关注一件事。
4. **视觉风格**：简洁、克制、留白充足，流式输出（打字机式实时展示 AI 思考），整体向 Claude 的清爽观感靠拢。
5. **可成长**：板块化结构天然适合后续把其他产品线作为新板块逐个接入，与"分阶段重构"的节奏完美契合。

> 这一原则同时取代了旧库复杂的"文件夹式 Navbar"，是本期界面设计的最高准则。

## 4. 关键体验要求（重写时必须正确处理的「热点」）

这些来自旧库的硬经验，重写客户端时**必须照做**（详见分析文档第 8 节）：

1. **实时流式**：分析过程用流式推送实时展示，不能让用户干等白屏。
2. **降级要透明**：某个 AI 专家超时返回了「兜底结果」，界面要标注「部分结果/置信度低」，不能假装一切正常。
3. **相似论文可溯源**：每篇相似论文能点击跳到原文链接，标注权威等级。
4. **辩论与仲裁透明**：是否触发辩论、为什么、最终怎么加权裁决，都要让用户看得到。
5. **缓存要提示**：命中缓存的报告要标「来自缓存」，并提供「刷新重新生成」。
6. **超时不崩**：整体有时间预算，超时也要返回已完成部分（`isPartial`），不能白屏死亡。

## 5. 技术选型（保持稳妥，沿用经过验证的栈）

> 大白话：用旧产品已经验证可行的同一套技术来重写，少踩坑、上线快；只是把代码结构重新设计得更干净。

| 方面 | 选型 | 理由 |
|---|---|---|
| 形态 | 网页应用（浏览器打开网址，手机电脑都能用） | 用户免安装，迁移最顺 |
| 前端 + 后端框架 | Next.js 14（App Router）+ React 18 + TypeScript | 与旧库同栈，可对照参考；前后端一体，部署简单 |
| 样式 | Tailwind CSS | 与旧库一致 |
| 数据库/云端 | Supabase（PostgreSQL） | 与旧库一致，缓存与未来账号体系都靠它 |
| AI 模型 | DeepSeek（含 R1）/ Minimax / Moonshot 多模型容错降级 | 与旧库一致 |
| 部署 | Vercel | 与旧库一致 |
| 包管理 | npm | 简单稳妥 |

> 后续性能优化项（非本期）：Rust WASM 质检引擎、Python LangGraph 引擎。

## 6. 新工程的代码结构（设计目标：比旧库更清晰）

```
src/
  app/
    layout.tsx            ← 全局布局（左侧板块导航栏 + 主区域）
    page.tsx              ← 默认进入「创新分析」板块
    report/[id]/          ← 报告页（可独立分享）
    api/
      analyze/            ← 核心分析接口（SSE）
      dual-track/         ← 双轨检索接口（可内联）
  core/                   ← 业务核心（替代旧 agents/，结构更清晰）
    orchestrator/         ← 分层调度
    agents/               ← 各 Agent（学术/产业/竞品/创新/跨域/仲裁/质检）
    ai-client/            ← 多模型客户端（信号量/降级/退避）
    search/               ← 双轨检索（academic/ industry/）
  components/
    shell/                ← Claude Desktop 式外壳：左侧板块导航栏 + 主区域容器
    boards/               ← 各功能「板块」（本期只激活 analysis 板块，其余占位）
      analysis/           ← 创新分析板块：输入 / 分析中 / 报告 三态
    report/               ← 报告渲染（单一引擎：雷达图/评分/证据/透明化）
    ui/                   ← 通用基础组件（简洁克制风格）
  lib/
    supabase/             ← 数据库封装
    types/                ← 全局类型定义
docs/                     ← 计划与进度（已建）
```

## 8. 验收标准（怎么算本期完成）

1. 在浏览器打开本地/部署网址，能看到首页。
2. 输入一个创意点提交，能看到实时分析进度（多个 Agent 状态流动）。
3. 分析结束能看到完整报告：评分 + 6 维雷达图 + 关键发现 + 相似论文（可点） + 证据 + 仲裁透明化。
4. 相同创意短期内重查命中缓存并有标注。
5. 任一 AI 专家超时，界面有「部分结果」标注且不崩溃。
6. 全流程在合理时间预算内完成，超时返回 `isPartial` 部分结果。

## 9. 已确认决策（2026-06-09 定稿）

| 问题 | 决策 |
|---|---|
| 目标用户 | **专业人群**：创业者/PM、科研/技术、投资/分析师 |
| 核心价值 | **AI 分析够深、结论服人**（可信度优先） |
| 用户输入 | **长文先行**；文档上传/链接解析作为本期增量子阶段 7.5 |
| 报告重点 | **证据溯源 + 反方/不确定性 + 量化评分/雷达图**（"可执行下一步建议"本期不重点做） |
| 是否登录 | **免登录**，打开网址即可直接试用（账号/付费体系列入第二阶段） |
| 分析模式 | **本期只做标准（深度）模式**，极速模式后续再加 |
| 界面语言 | **中英双语**（界面提供中/英切换；i18n 结构从一开始就搭好） |
| 产品名 | **沿用 Novoscan** |
| 形态 | 网页应用（浏览器打开网址，免安装） |
| 重写程度 | 全栈重写，含 Agent 核心 |
| 界面准则 | Claude Desktop 式分板块（见 §3.5） |

> 据此，§2 范围相应收敛：本期不做极速模式、不做登录；§3.5 板块导航本期只激活「创新分析」一个板块。
